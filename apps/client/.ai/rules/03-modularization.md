# Modularization Rules (Client)

## Component Organization
- Organize components in logical groups (atoms, molecules, organisms)
- Keep components small and focused
- Use proper file naming conventions (PascalCase for components)

## Reusability
- Extract common UI patterns into reusable components
- Use composition over inheritance
- Create shared hooks for repeated logic

## Maintainability
- Write self-documenting code with clear intent
- Keep components loosely coupled
- Use dependency injection via context when needed
